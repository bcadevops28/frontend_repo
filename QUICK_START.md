# Quick Start Guide

## Start the Frontend

```bash
npm install
npm start
```

The app opens at `http://localhost:3000`

## Expected Spring Boot Backend Endpoints

```
GET    /api/courtcases
GET    /api/courtcases/{id}
POST   /api/courtcases
PUT    /api/courtcases/{id}
DELETE /api/courtcases/{id}
```

## Court Case Model

```javascript
{
  id: number,                           // Auto-generated by backend
  caseNumber: string,                   // Unique case identifier
  categoryType: "Criminal" | "Civil",   // Type of case
  description: string,                  // Case details (min 10 chars)
  status: "Pending" | "In Progress" | "Closed"  // Case status
}
```

## Modify API URL

Edit [src/services/courtCaseService.js](src/services/courtCaseService.js) line 3:

```javascript
const API_BASE_URL = 'http://your-backend-url/api/courtcases';
```

## Enable CORS on Backend

Add to your Spring Boot application:

```java
@Configuration
public class CorsConfig {
    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/api/**")
                    .allowedOrigins("http://localhost:3000")
                    .allowedMethods("GET", "POST", "PUT", "DELETE");
            }
        };
    }
}
```

## Build for Production

```bash
npm run build
```

Output: `build/` directory ready for deployment

## Common Issues & Fixes

| Issue | Solution |
|-------|----------|
| "Backend not responding" | Check if Spring Boot is running on `http://localhost:8080` |
| CORS error | Configure CORS in Spring Boot backend |
| Form validation fails | Ensure description is at least 10 characters |
| Data not loading | Click "Refresh" button or check browser console |
